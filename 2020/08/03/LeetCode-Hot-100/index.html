LeetCode Hot 100

使用方式（Android / Java / 现场手撕）

  • 计时：Medium 25min / Hard 45min；超时先看思路，再关掉题解重写一遍
  • 每题记录：类型 + 核心不变量 + 边界用例 + 复杂度
  • 复盘：错题按 +1/+3/+7 天闭卷重写

两周核心稳态计划（推荐）

每天固定三件事：

  1. 新题 4 题（限时写完；不会就看思路后关掉题解重写）
  2. 复盘 1 题（闭卷重写）：优先选昨天做错/卡住的题；没错题就复写昨天最耗时的题
  3. 工程型手撕（隔天 1 次，30-45min）：按 Day 2/4/6/8/10/12/14 做 1 个（见下方附录）

新题安排（14 天）：

  • Day 6（区间/堆）：56 合并区间、215 数组中的第K个最大元素、253 会议室 II、347 前 K 个高频元素
  • Day 7（链表基础）：19 删除链表的倒数第 N 个结点、21 合并两个有序链表、141 环形链表、206 反转链表

工程型手撕安排（隔天 1 个，30-45min）：

  • Day 2：LRU Cache
  • Day 4：阻塞队列：生产者-消费者
  • Day 6：重试器：指数退避 + 最大次数 + 可取消
  • Day 8：限流器：Token Bucket
  • Day 10：带超时的缓存：LRU + TTL
  • Day 12：Debounce/Throttle
  • Day 14：线程安全单例：双检锁

附录：工程型手撕（Java / Android 常见）

面试现场经常会从业务/工程抽象出“写个类/写个工具”。建议你至少能在白板/纸上写出核心版本，并解释线程安全和复杂度。

  • LRU Cache（可先写非线程安全；再说怎么加锁/分段锁/读写锁）
  • 带超时的缓存：LRU + TTL（如何处理过期清理、惰性删除）
  • 阻塞队列：生产者-消费者（ReentrantLock + Condition 或 wait/notify）
  • 重试器：指数退避 + 最大次数 + 可取消（线程中断语义）
  • 限流器：Token Bucket（时间推进、并发安全）
  • Debounce/Throttle（ScheduledExecutorService，支持取消）
  • 线程安全单例：双检锁（volatile + synchronized，解释内存可见性）
